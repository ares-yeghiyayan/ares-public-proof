# Seven-Day GitHub Balance Plan

This plan is for making the founder GitHub profile look balanced, credible, and active without exposing private project internals. It should be executed manually after the private repo owner decides the public scope is safe.

## Target Public Activity Mix

```text
45-55% commits
20-25% pull requests
15-20% issues
10-15% reviews
```

## Day 1: Create GitHub Profile README Repo

Create the public profile repository `ares-yeghiyayan/ares-yeghiyayan`. Copy `public_starter_kit/profile_README.md` into that repository as `README.md`.

Keep the profile serious, founder-led, technical, and non-hype.

## Day 2: Create Public Proof Repo With Sanitized README

Create the public repository `ares-public-proof`. Copy `public_starter_kit/founder_public_proof_README.md` into that repository as `README.md`.

Do not copy private files directly.

## Day 3: Add Public Proof Manifest and Safe Scope

Copy the public proof manifest and safe public scope files into the public proof repo. Use them as the boundary before adding any other public material.

## Day 4: Add One Public-Safe Learning Note or Build-Rhythm Note

Add one sanitized learning note or build-rhythm note. Keep it focused on discipline, review habits, documentation habits, or testing habits.

Do not publish diagrams, module summaries, private naming systems, sensitive assumptions, prompts, or private implementation details.

## Day 5: Open One Issue and One PR in the Public Repo

Open one public issue describing a small documentation improvement. Open one PR that closes that issue. This creates visible execution discipline without pretending the public repo is the product.

## Day 6: Add One Small CI or Markdown Quality Check

Add one lightweight check, such as markdown linting or a simple link/documentation check. The goal is proof discipline, not product behavior.

## Day 7: Pin Repositories and Review Profile Balance

Pin the profile README repo and public proof repo. Review the profile for balance across commits, pull requests, issues, and reviews.

Check again for safety: no financial advice, no trading claims, no customer data, no live trading, no broker execution, no performance claims, no secrets, and no proprietary internals.
