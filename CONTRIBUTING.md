# Contributing

This repository is a public proof surface for learning discipline, documentation rhythm, review habits, and small safe improvements.

Contributions should keep the public surface narrow, high-level, and non-sensitive.

## What Belongs Here

Good contributions include:

- public-safe learning notes
- documentation improvements
- reusable note templates
- checklists that improve review discipline
- CI or markdown quality checks
- changelog updates
- small issue and pull request workflow improvements

## What Does Not Belong Here

Do not add:

- private architecture
- internal names
- product mechanics
- detailed workflows
- customer strategy
- market positioning
- private prompts
- proprietary methods
- broker or execution details
- customer data
- performance claims
- financial advice claims
- production readiness claims

## Contribution Rhythm

Use a small, reviewable workflow:

1. Open a clear issue.
2. Create a focused branch.
3. Make one narrow change.
4. Keep the change easy to review.
5. Confirm the safety boundary.
6. Keep CI passing.
7. Merge only after the change remains public-safe.

## Pull Request Checklist

Before opening or merging a pull request, check:

- [ ] The change is small and reviewable.
- [ ] The change is public-safe.
- [ ] No private architecture or internal names are exposed.
- [ ] No customer strategy, market positioning, or proprietary methods are exposed.
- [ ] No performance, financial advice, or production readiness claims are made.
- [ ] CI remains passing.
- [ ] The changelog is updated when useful.

## Review Standard

A good public contribution should answer this question:

> What did this improve publicly without expanding the private exposed surface area?
